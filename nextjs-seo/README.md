# Next.js SEO Skills

Skills for optimizing Next.js applications for search engines.

## Overview

These skills provide practical guidance for web development tasks, from SEO optimization to performance tuning and best practices.

## Skills in This Category

### [nextjs-seo](./nextjs-seo)
Optimize Next.js applications for search engines using JSON-LD structured data, metadata API, and layout best practices. Implements schema.org types for rich search results.

**When to use**: Building or optimizing Next.js sites for search visibility, implementing structured data, setting up proper metadata

**Key features**: JSON-LD implementation, Metadata API, schema.org types, Open Graph tags, validation tools

## Why Next.js SEO Skills Matter

**Search Visibility = Traffic**
Proper SEO implementation can increase organic traffic by 2-10x

**Rich Results = Higher CTR**
Structured data enables rich search results (ratings, prices, events) that get 30-40% higher click-through rates

**Technical SEO = Foundation**
Metadata, structured data, and proper architecture form the foundation of search visibility

## Quick Start Guide

### For a New Next.js Project

1. Use [nextjs-seo](./nextjs-seo) skill
2. Set up root layout with metadata base and title template
3. Implement JSON-LD on homepage (WebSite schema)
4. Add Organization schema to about page
5. Validate with Google Rich Results Test

### For Existing Projects

1. Audit current SEO implementation
2. Identify missing metadata and structured data
3. Use [nextjs-seo](./nextjs-seo) skill to add schema types
4. Test with validation tools
5. Monitor in Google Search Console

## Common Use Cases

### E-commerce Site
- Product schema on product pages
- Organization schema on about page
- Breadcrumb navigation schema
- Review/Rating aggregation

### Content/Blog Site
- Article schema on blog posts
- Author/Person schema
- WebSite schema with search action
- Image object schema

### SaaS Product
- Organization schema
- SoftwareApplication schema
- FAQ schema for documentation
- Video schema for demos

## Validation Tools

Always validate your implementation:

1. **Google Rich Results Test**: [https://search.google.com/test/rich-results](https://search.google.com/test/rich-results)
2. **Schema Markup Validator**: [https://validator.schema.org/](https://validator.schema.org/)
3. **Google Search Console**: Monitor actual search performance

## SEO Impact Timeline

**Week 1-2**: Implementation complete, validated
**Week 3-4**: Google starts crawling new structured data
**Month 2-3**: Rich results begin appearing
**Month 3-6**: Measurable traffic increases from better visibility

## Best Practices Across All Skills

1. **Validate everything**: Use Google's Rich Results Test before deploying
2. **Start with high-value pages**: Homepage, product pages, main content
3. **Monitor performance**: Use Search Console to track rich results
4. **Update regularly**: Keep structured data in sync with page content
5. **Follow Next.js patterns**: Use the Metadata API, not manual meta tags
